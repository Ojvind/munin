Grund

Infrastruktur & beroenden

Projektet fick ett rejält lyft i teknisk bas innan nya features kunde byggas.

Frontend (Munin)

  • React uppgraderat till 18.3
  • @mui/styles → @mui/material/styles
  • ESLint konfigurerat till airbnb v19
  • react/require-default-props avstängt — kolliderar med React 18.3
  • Oanvända komponenter och console.log städat

Backend (Yggdrasil)

  • Docker Compose med Caddy som reverse proxy + automatisk HTTPS
  • MinIO för bildlagring med eget subdomän
  • Separata dev/prod-ports och compose-filer
  • MongoDB-port stängd externt i prod
  • Seed-script med 15 författare och hundratals böcker
Yggdrasil

Säkerhet & autentisering

API:et gick från i princip öppet till fullständigt skyddat med JWT och rollbaserad behörighet.

  • Alla GraphQL-resolvers skyddas med isAuthenticated via graphql-resolvers
  • Raderingsoperationer kräver dessutom isAdmin
  • JWT-token skickas som Authorization: Bearer-header vid varje request
  • GraphQL introspection inaktiverat i produktion — schemat exponeras inte utåt
  • Inloggningsfel ger ett generiskt meddelande — förhindrar username enumeration
  • Admin-bootstrapping: signUp accepterar adminSecret vid första uppstart
  • Applikationen vägrar starta utan SECRET-variabeln (fail fast)
  • På klienten: utloggning rensar JWT och Apollo-cachen (client.clearStore())
Data & lagring

Bildhantering & MinIO

Porträtt och bokomslag fick en genomtänkt och skalbar hantering.

  • Bilder lagras i MinIO (S3-kompatibelt) och exponeras via eget subdomän
  • Databasen sparar bara filnamnet — URL:en byggs från miljövariabel REACT_APP_IMAGES_URL
  • Standardbilder visas automatiskt när foto saknas (författare och böcker)
  • Porträtten i listorna renderas som kvadratiska thumbnails med hover-preview
  • Uppladdning direkt i edit-formulären via useImageUpload-hook
Design

Designsystem & teman

Från hårdkodad mörkgrön till ett fullständigt dynamiskt temasystem.

Earth — varm beige Ocean — stålblå Dark — lila accent
  • Alla färger definierade som CSS custom properties i _theme-vars.scss
  • data-theme sätts på body — inga färger i komponenterna
  • Temavalet sparas i localStorage och återställs vid nästa besök
  • Navigation, footer, DataGrid, kort och detaljvyer — allt följer aktivt tema
  • Temaväljarens ikoner (berg, vågor, måne) med MUI Tooltip direkt vid hover
Lokalisering

Tre språk — fullt genomfört

Hela UI:t är översatt till italienska, svenska och engelska med react-i18next.

  • Navigering, formulärfält, bekräftelsedialoger, felmeddelanden, temaknappar
  • Landsflaggor i headern — klick byter språk och sparas i localStorage
  • På mobil: native <select> istället för flaggor (inget hover på touch)
  • CSS Grid (1fr auto 1fr) förhindrar layout shift när text ändrar längd vid språkbyte
  • Alla tre lokaler lever i separata JSON-filer och är komplett täckta
UX

Mobilanpassning

Applikationen är byggd för att fungera bra både på desktop och mobil.

Navigation

  • Breakpoint utökat till 768 px
  • Språk- och temabyte som native <select> på mobil
  • Grid-layout hindrar knappar att hoppa vid textbredd-ändring
  • white-space: nowrap på nav-länkarna

Listor

  • DataGrid ersätts av kortbaserad mobilvy under 600 px
  • Separate BookMobileCard och WriterMobileCard
  • Korten följer aktivt tema via CSS custom properties
  • Obegränsat antal rader — ingen pageSize-begränsning
Design

Författardetaljvy — kompakt kortdesign

Detaljkortet för en författare designades om från grunden, iterativt.

  • 120 px porträtt med object-fit: cover och hover-tooltip som visar bilden i originalformat och originalproportioner
  • Namn (h4), landsflagga (21 px) och hemsidelänk inline i en kompakt rad
  • Beskrivningstexten ligger full bredd under porträttraden — inga tomma kolumner
  • Redigera-ikon i övre hörnet — edit-läget bäddar in alla fält direkt i kortet
  • Beskrivningsrutan är vertikalt resizable med musen
Feature

Rich text-editor för beskrivningar

Enkla formateringsmöjligheter i beskrivningsfälten för både författare och böcker.

  • Bygger på Tiptap — headless, React 18-kompatibel, välunderhållen
  • Toolbar med B, I och U-knappar ovanför textfältet
  • Tangentbordsgenvägar: ⌘B, ⌘I, ⌘U fungerar automatiskt
  • Aktiv formatering markeras med highlight i toolbaren
  • Utdata är HTML — renderas med dangerouslySetInnerHTML
  • All HTML saniteras med DOMPurify innan rendering — XSS-skydd
  • Matchande utseende med övriga MUI outlined-fält
Yggdrasil

Datamodell & API-förbättringar

Flera genomgripande förändringar av datamodellen i Yggdrasil.

Beskrivning på Writer

  • Nytt fält description på Writer-modellen
  • Tillagt i GraphQL-schema, resolver, mutation och Munin-frontend
  • Radbrytningar bevaras med white-space: pre-wrap (och nu HTML)

Valfritt förnamn

  • name gjort icke-obligatoriskt — stödjer ennamnsförfattare som Epiktetos
  • Namn visas med filter(Boolean).join(' ') överallt
  • Ändrat i Mongoose-schema, GraphQL-typdefinition och Munin-mutations
Feature

Flera författare per bok

En av de större datamigrationerna — från ett enskilt writerId till en array.

  • Yggdrasil: writerId: ObjectId → writerIds: [ObjectId] i Mongoose-modellen
  • GraphQL-schema: writer: Writer! → writers: [Writer!]! på Book-typen
  • Resolver laddar alla författare parallellt via DataLoader
  • Databasmigrering via mongosh direkt i Docker-containern — befintliga böcker fick writerId kopierat till writerIds: [id]
  • Munin: MUI Autocomplete med multiple i edit-formuläret
  • Bokdetaljvyn visar alla författare kommaseparerade med egna RouterLinks
